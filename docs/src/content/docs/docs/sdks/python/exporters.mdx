The SideSeat Python SDK supports three exporter types. You can use multiple exporters simultaneously.

## OTLP Exporter

The primary exporter for sending traces to SideSeat or any OTLP-compatible endpoint.

### Basic Usage

```python
from sideseat import SideSeatTelemetry

telemetry = SideSeatTelemetry()
telemetry.setup_otlp_exporter()
```

By default, sends to `http://localhost:4318/v1/traces` (OpenTelemetry default).

### Custom Endpoint

```python
telemetry.setup_otlp_exporter(
    endpoint="http://localhost:5388/otel/default/v1/traces"
)
```

### With Authentication

```python
telemetry.setup_otlp_exporter(
    endpoint="http://localhost:5388/otel/default/v1/traces",
    headers={"Authorization": "Bearer sk-your-api-key"}
)
```

### With Compression

```python
from opentelemetry.exporter.otlp.proto.http import Compression

telemetry.setup_otlp_exporter(
    compression=Compression.Gzip
)
```

### Custom Timeout

```python
telemetry.setup_otlp_exporter(
    timeout=30  # seconds
)
```

### Environment Variables

The OTLP exporter respects these environment variables:

| Variable | Description | Default |
|----------|-------------|---------|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | Base endpoint URL | `http://localhost:4318` |
| `OTEL_EXPORTER_OTLP_TRACES_ENDPOINT` | Traces-specific endpoint | (base)/v1/traces |
| `OTEL_EXPORTER_OTLP_HEADERS` | Headers (key=value,key2=value2) | (none) |
| `OTEL_EXPORTER_OTLP_TIMEOUT` | Timeout in milliseconds | `10000` |
| `OTEL_EXPORTER_OTLP_COMPRESSION` | Compression algorithm | (none) |

## Console Exporter

Prints spans to stdout. Useful for debugging during development.

### Basic Usage

```python
telemetry = SideSeatTelemetry()
telemetry.setup_console_exporter()
```

### Output Example

```
{
    "name": "llm.completion",
    "context": {
        "trace_id": "0x1234567890abcdef...",
        "span_id": "0xabcdef1234567890"
    },
    "start_time": "2024-01-15T10:30:00.000000Z",
    "end_time": "2024-01-15T10:30:01.500000Z",
    "attributes": {
        "gen_ai.system": "openai",
        "gen_ai.request.model": "gpt-4",
        "gen_ai.usage.input_tokens": 150,
        "gen_ai.usage.output_tokens": 200
    }
}
```

<Aside type="tip">
Combine with OTLP for development:
```python
telemetry.setup_otlp_exporter()
telemetry.setup_console_exporter()  # See spans in terminal too
```
</Aside>

## File Exporter

Writes spans to a JSONL (JSON Lines) file. Each line is a complete JSON object.

### Basic Usage

```python
telemetry = SideSeatTelemetry()
telemetry.setup_file_exporter("traces.jsonl")
```

### Append Mode (Default)

```python
# Append to existing file
telemetry.setup_file_exporter("traces.jsonl", mode="a")
```

Each run adds new spans to the file.

### Overwrite Mode

```python
# Overwrite on each run
telemetry.setup_file_exporter("traces.jsonl", mode="w")
```

Clears the file at startup.

### Custom Path

```python
import os
from pathlib import Path

# Absolute path
telemetry.setup_file_exporter("/var/log/myapp/traces.jsonl")

# Relative to home
home = Path.home()
telemetry.setup_file_exporter(str(home / ".myapp" / "traces.jsonl"))

# Date-based
from datetime import datetime
filename = f"traces-{datetime.now().strftime('%Y-%m-%d')}.jsonl"
telemetry.setup_file_exporter(filename)
```

### Output Format

```json
{"name":"llm.completion","trace_id":"abc123...","span_id":"def456...","parent_span_id":null,"start_time":"2024-01-15T10:30:00.000000Z","end_time":"2024-01-15T10:30:01.500000Z","attributes":{"gen_ai.system":"openai"}}
{"name":"tool.call","trace_id":"abc123...","span_id":"ghi789...","parent_span_id":"def456...","start_time":"2024-01-15T10:30:01.000000Z","end_time":"2024-01-15T10:30:01.200000Z","attributes":{"tool.name":"calculator"}}
```

### Binary Data Handling

When `encode_binary_as_base64=True` (default), binary data in attributes is base64-encoded:

```python
# Binary attribute
span.set_attribute("image.data", b"\x89PNG\r\n...")

# In file
{"attributes": {"image.data": "iVBORw0KGgo..."}}
```

## Multiple Exporters

Use multiple exporters for redundancy or different use cases:

```python
telemetry = SideSeatTelemetry()

# Primary: SideSeat server
telemetry.setup_otlp_exporter(
    endpoint="http://localhost:5388/otel/default/v1/traces"
)

# Backup: Local file
telemetry.setup_file_exporter("traces-backup.jsonl")

# Debug: Console output (development only)
if os.getenv("DEBUG"):
    telemetry.setup_console_exporter()
```

## Custom Exporters

You can use any OpenTelemetry-compatible exporter by providing your own `TracerProvider`:

```python
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from my_custom_exporter import CustomExporter

# Create provider with custom exporter
provider = TracerProvider()
provider.add_span_processor(BatchSpanProcessor(CustomExporter()))

# Pass to SideSeatTelemetry
telemetry = SideSeatTelemetry(tracer_provider=provider)

# Can still add more exporters
telemetry.setup_file_exporter("traces.jsonl")
```

## Exporter Comparison

| Exporter | Use Case | Persistence | Latency |
|----------|----------|-------------|---------|
| OTLP | Production, real-time monitoring | Remote server | Low |
| Console | Development, debugging | None | Instant |
| File | Backup, offline analysis | Local file | Low |

## Best Practices

1. **Production**: Use OTLP exporter only (minimal overhead)
2. **Development**: OTLP + Console for visibility
3. **CI/CD**: File exporter for artifact collection
4. **High-volume**: Consider sampling when using multiple exporters

```python
# Production setup
telemetry = SideSeatTelemetry().setup_otlp_exporter()

# Development setup
telemetry = (SideSeatTelemetry()
    .setup_otlp_exporter()
    .setup_console_exporter())

# CI/CD setup
telemetry = SideSeatTelemetry().setup_file_exporter("traces.jsonl")
```
